# Устранение неполадок

## Обзор

В этом разделе описаны распространенные проблемы и способы их решения при эксплуатации приложения VoxLibris. Здесь также представлены рекомендации по диагностике и устранению различных неполадок.

## Диагностика проблем

### Проверка состояния сервисов

Для начала диагностики проверьте состояние всех компонентов системы:

```bash
# Проверка состояния приложения
pm2 status

# Проверка состояния базы данных
sudo systemctl status postgresql

# Проверка состояния MinIO (если используется)
sudo systemctl status minio

# Проверка используемых портов
sudo netstat -tulpn | grep :5000
sudo netstat -tulpn | grep :3000
sudo netstat -tulpn | grep :9000
```

### Проверка логов

Проверка логов - первый шаг к диагностике:

```bash
# Логи приложения (если используется PM2)
pm2 logs voxlibris-app

# Логи PostgreSQL
sudo tail -f /var/log/postgresql/postgresql-14-main.log

# Логи MinIO
sudo journalctl -u minio -f

# Системные логи
sudo journalctl -u voxlibris-app -f
```

## Распространенные проблемы

### Проблемы с подключением к базе данных

**Симптомы:**
- Приложение не запускается
- Ошибка: "FATAL: password authentication failed for user"
- Ошибка: "connection refused"

**Решение:**
1. Проверьте правильность строки подключения в `.env`:
   ```
   DATABASE_URL="postgresql://username:password@host:port/database_name"
   ```
2. Убедитесь, что PostgreSQL запущен:
   ```bash
   sudo systemctl status postgresql
   sudo systemctl start postgresql
   ```
3. Проверьте, что пользователь существует и имеет доступ:
   ```bash
   sudo -u postgres psql
   \du  # Показать всех пользователей
   \l   # Показать все базы данных
   ```

### Проблемы с аутентификацией

**Симптомы:**
- Невозможно войти в систему
- Ошибки JWT токенов
- "Invalid token" или "Unauthorized" ответы

**Решение:**
1. Проверьте, что JWT секреты установлены корректно:
   ```
   JWT_SECRET="valid_secret_with_min_32_chars"
   JWT_REFRESH_SECRET="valid_refresh_secret_with_min_32_chars"
   ```
2. Убедитесь, что время на сервере синхронизировано:
   ```bash
   sudo ntpdate -s time.nist.gov
   ```
3. Проверьте логи на наличие ошибок аутентификации

### Проблемы с файловым хранилищем

**Симптомы:**
- Ошибка при загрузке книг или изображений
- "Upload failed" сообщения
- Ошибки доступа к файлам

**Решение:**
1. Проверьте права доступа к директориям:
   ```bash
   ls -la /var/lib/voxlibris/uploads/
   sudo chown -R $USER:$USER /var/lib/voxlibris/
   sudo chmod -R 755 /var/lib/voxlibris/
   ```
2. Для S3/MinIO - проверьте конфигурацию:
   ```
   S3_ENDPOINT=http://localhost:9000
   S3_ACCESS_KEY_ID=minioadmin
   S3_SECRET_ACCESS_KEY=minioadmin
   S3_BUCKET_NAME=voxlibris-bucket
   ```
3. Убедитесь, что MinIO/Сервер S3 запущен и доступен

### Проблемы с производительностью

**Симптомы:**
- Медленная загрузка страниц
- Задержки при воспроизведении аудио
- Высокое использование CPU или памяти

**Решение:**
1. Проверьте использование ресурсов:
   ```bash
   top
   htop
   df -h
   ```
2. Проверьте логи на наличие медленных запросов
3. Включите логирование медленных запросов PostgreSQL:
   ```
   # В postgresql.conf
   log_min_duration_statement = 1000  # логировать запросы дольше 1 секунды
   ```
4. Оптимизируйте запросы и добавьте индексы при необходимости

## Проблемы с Docker

### Контейнер не запускается

**Симптомы:**
- Ошибка при запуске Docker контейнера
- Контейнер постоянно перезапускается

**Решение:**
1. Проверьте логи контейнера:
   ```bash
   docker logs <container-id>
   docker-compose logs app
   ```
2. Проверьте переменные окружения:
   ```bash
   docker-compose config  # Проверить конфигурацию
   ```
3. Проверьте зависимости (база данных, хранилище):
   ```bash
   docker-compose ps  # Проверить статус сервисов
   ```

### Проблемы с сетью

**Симптомы:**
- Приложение внутри контейнера не может подключиться к базе данных
- Ошибки подключения к внешним сервисам

**Решение:**
1. Убедитесь, что сервисы находятся в одной сети:
   ```yaml
   # docker-compose.yml
   services:
     app:
       networks:
         - app-network
     postgres:
       networks:
         - app-network
         
   networks:
     app-network:
       driver: bridge
   ```
2. Проверьте, что используете имя сервиса в качестве хоста:
   ```bash
   # Вместо localhost:5432 используйте postgres:5432
   DATABASE_URL="postgresql://user:pass@postgres:5432/dbname"
   ```

## Проблемы с миграциями

### Ошибка при выполнении миграций

**Симптомы:**
- Ошибка при запуске `pnpm run db:migrate`
- Ошибка: "Migration failed"

**Решение:**
1. Проверьте, что база данных доступна
2. Проверьте, что миграции корректны:
   ```bash
   # Посмотреть статус миграций
   pnpm run db:validate
   ```
3. При необходимости сбросьте миграции (ТОЛЬКО в разработке!):
   ```bash
   # Сброс базы данных (только в разработке!)
   pnpm run db:reset
   ```

### Конфликт миграций

**Симптомы:**
- Ошибка: "duplicate migration name"
- Ошибка: "migration already applied"

**Решение:**
1. Проверьте таблицу миграций:
   ```sql
   SELECT * FROM drizzle_migrations ORDER BY id DESC;
   ```
2. При необходимости удалите некорректные записи:
   ```sql
   DELETE FROM drizzle_migrations WHERE id = <problematic_id>;
   ```
3. Проверьте файлы миграций на дубликаты имен

## Проблемы с WebSocket

### Ошибки подключения к WebSocket

**Симптомы:**
- Ошибки при подключении к сессиям чтения
- "WebSocket connection failed"
- Проблемы с синхронизацией в реальном времени

**Решение:**
1. Проверьте, что порт для WebSocket открыт
2. Убедитесь, что nginx настроен корректно:
   ```nginx
   location /ws {
       proxy_pass http://localhost:5000;
       proxy_http_version 1.1;
       proxy_set_header Upgrade $http_upgrade;
       proxy_set_header Connection "upgrade";
       proxy_set_header Host $host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
   }
   ```
3. Проверьте настройки CORS в приложении

## Проблемы с электронной почтой

### Ошибки отправки писем

**Симптомы:**
- Письма не отправляются
- Ошибки SMTP

**Решение:**
1. Проверьте настройки SMTP:
   ```
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=your_email@gmail.com
   SMTP_PASS=your_app_specific_password
   ```
2. Для Gmail убедитесь, что включены "App passwords"
3. Проверьте, что порт не заблокирован хостинг-провайдером

## Рекомендации по устранению неполадок

1. **Всегда начинайте с проверки логов** - они содержат наиболее важную информацию
2. **Изолируйте проблему** - определите, какой компонент вызывает проблему
3. **Проверьте зависимости** - убедитесь, что все сервисы доступны
4. **Проверьте конфигурацию** - часто проблема в неправильных настройках
5. **Используйте тестовую среду** - воспроизводите и тестируйте решения в изолированной среде
6. **Создавайте резервные копии** - перед внесением изменений создавайте резервные копии
7. **Документируйте решения** - заносите найденные решения в эту документацию
8. **Обновляйте систему** - регулярно обновляйте зависимости и компоненты
9. **Проверяйте права доступа** - часто проблема в неправильных разрешениях
10. **Следите за ресурсами** - недостаток памяти или дискового пространства может вызывать проблемы
