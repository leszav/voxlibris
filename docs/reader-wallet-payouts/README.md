# Демо-кошелек чтеца и payout layer

**Статус:** Roadmap  
**Дата:** 2026-06-29  
**Связанные документы:** [Монетизация](../MONETIZATION.md), [Клубы чтецов](../02-architecture/reader-clubs-system.md)

## Назначение

Этот пакет документов описывает демонстрационный кошелек чтеца и будущий слой выплат.

Цель demo-slice - показать цепочку:

```text
платеж слушателя -> ledger-начисление чтецу -> баланс -> демо-заявка на вывод
```

При этом demo-режим не выполняет реальных выплат, не вызывает банк, ФНС или боевые API ЮKassa и не должен создавать бухгалтерских обязательств.

## Ключевое решение

`commerce_ledger_entries` остается источником правды по начислениям:

- `reader_earning` - доход чтеца;
- `platform_fee` - комиссия платформы;
- `acquiring_fee` - комиссия эквайринга.

Демо-кошелек не пересчитывает платежи заново. Он только агрегирует уже созданные ledger entries и создает отдельные payout-заявки с `provider_code = 'demo'`.

## Что не входит в demo-slice

- реальные выплаты самозанятым;
- проверка статуса самозанятого;
- KYC/AML;
- фискальные чеки выплат;
- интеграция с ЮKassa Payouts или банком;
- автоматическое закрытие бухгалтерского периода.
