# Система уведомлений и почты

## Обзор

Модуль системы уведомлений и почты - это важная часть приложения VoxLibris, обеспечивающая взаимодействие с пользователями вне активной сессии. Система включает в себя отправку электронной почты, push-уведомления и внутренние уведомления в приложении.

## Компоненты системы

### 1. Email Service ([server/services/email-service.ts](file:///home/odmen/DEV/voxlibris/server/services/email-service.ts))

Сервис для отправки электронных писем:

- Отправка подтверждений регистрации
- Отправка приглашений в клубы
- Отправка уведомлений о событиях
- Восстановление паролей
- Уведомления о модерации

### 2. Notification Service ([server/services/notification-service.ts](file:///home/odmen/DEV/voxlibris/server/services/notification-service.ts))

Сервис для внутренних уведомлений:

- Создание уведомлений в базе данных
- Управление статусами уведомлений
- Отправка push-уведомлений
- Интеграция с другими системами

### 3. Notifications Routes ([server/routes/notifications.ts](file:///home/odmen/DEV/voxlibris/server/routes/notifications.ts))

Маршруты для работы с уведомлениями:

- `/api/notifications` - получение списка уведомлений
- `/api/notifications/unread` - получение непрочитанных уведомлений
- `/api/notifications/:id/read` - отметка уведомления как прочитанного
- `/api/notifications/settings` - настройки уведомлений пользователя

## Архитектура системы уведомлений

### Типы уведомлений

Система поддерживает несколько типов уведомлений:

1. **Email-уведомления**:
   - Отправляются на email пользователя
   - Используются для важных событий
   - Требуют внешнего SMTP-сервера

2. **Push-уведомления**:
   - Отправляются через браузер или мобильное приложение
   - Требуют активного подключения
   - Более интерактивные

3. **Внутренние уведомления**:
   - Хранятся в приложении
   - Отображаются в интерфейсе
   - Имеют статус прочтения

### Категории уведомлений

Уведомления классифицируются на категории:

1. **Системные уведомления**:
   - Подтверждения регистрации
   - Уведомления о безопасности
   - Технические уведомления

2. **Социальные уведомления**:
   - Приглашения в клубы
   - Упоминания в чатах
   - Ответы на сообщения

3. **Событийные уведомления**:
   - Начало сессий чтения
   - Изменения в клубах
   - Доступные новые книги

4. **Персонализированные уведомления**:
   - Рекомендации книг
   - Прогресс чтения
   - Аналитика активности

## Основные функции

### Отправка email

Система обеспечивает отправку различных типов писем:

1. **Подтверждение регистрации**:
   - Генерация токена активации
   - Создание ссылки активации
   - Отправка персонализированного письма

2. **Приглашения в клубы**:
   - Генерация уникального токена приглашения
   - Вставка информации о клубе
   - Персонализация для приглашаемого

3. **Восстановление пароля**:
   - Генерация токена сброса
   - Безопасная ссылка для сброса
   - Временные ограничения

4. **Уведомления о событиях**:
   - Изменения в клубах
   - Новые книги
   - Начало сессий

### Управление внутренними уведомлениями

Система внутренних уведомлений обеспечивает:

1. **Создание уведомлений**:
   - Запись в базу данных
   - Назначение получателей
   - Классификация по типам

2. **Получение уведомлений**:
   - Фильтрация по категориям
   - Пагинация для производительности
   - Сортировка по времени

3. **Управление статусами**:
   - Отметка как прочитанное
   - Массовые изменения статуса
   - Архивирование старых уведомлений

### Настройки уведомлений

Пользователи могут настраивать получение уведомлений:

1. **Выбор каналов**:
   - Email
   - Push-уведомления
   - Внутренние уведомления

2. **Выбор категорий**:
   - Какие типы уведомлений получать
   - Частота отправки
   - Временные рамки

## Безопасность

- Защита от спама и рассылки нежелательных писем
- Проверка адресов получателей
- Логирование отправки уведомлений
- Ограничения на частоту отправки

## Возможные проблемы и решения

### Проблемы с доставкой

- **Недоступный email**: проверка валидности адресов
- **Блокировка писем**: соблюдение стандартов и практик
- **Ошибки отправки**: повторные попытки и резервное хранение

### Проблемы с производительностью

- **Большое количество уведомлений**: очередь отправки
- **Массовые рассылки**: пакетная обработка
- **Активные соединения**: оптимизация push-уведомлений

### Проблемы с пользователями

- **Нежелательные уведомления**: настройки и отказ от подписки
- **Смешение важных и неважных уведомлений**: приоритезация
- **Перегрузка информацией**: ограничение количества уведомлений

## Тестирование

Модуль уведомлений должен быть протестирован на:

- Корректность отправки различных типов писем
- Обработку ошибок отправки
- Безопасность и защиту от спама
- Интеграцию с другими частями приложения
- Обработку настроек пользователей
- Производительность при массовой отправке
