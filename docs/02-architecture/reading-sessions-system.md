# Система чтения и сессий

## Обзор

Модуль системы чтения и сессий - это ключевой компонент приложения VoxLibris, обеспечивающий синхронизацию чтения, управление сессиями, отслеживание прогресса и взаимодействие между участниками во время чтения. Система поддерживает как индивидуальное чтение, так и совместные сессии в клубах.

## Компоненты системы

### 1. Reading Sessions Routes ([server/routes/reading-sessions.ts](file:///home/odmen/DEV/voxlibris/server/routes/reading-sessions.ts))

Маршруты для управления сессиями чтения:

- `/api/reading-sessions` - создание и получение списка сессий
- `/api/reading-sessions/:id` - получение информации о сессии
- `/api/reading-sessions/:id/join` - присоединение к сессии
- `/api/reading-sessions/:id/leave` - выход из сессии
- `/api/reading-sessions/:id/progress` - обновление прогресса чтения
- `/api/reading-sessions/:id/control` - управление воспроизведением (для модераторов)

### 2. WebSocket Reading Sessions ([server/websocket/reading-sessions.ts](file:///home/odmen/DEV/voxlibris/server/websocket/reading-sessions.ts))

Обработчик WebSocket для реального времени в сессиях чтения:

- Синхронизация прогресса между участниками
- Управление воспроизведением (воспроизведение, пауза, перемотка)
- Отправка уведомлений о действиях участников
- Отслеживание присутствия участников

### 3. WebSocket Reader ([server/websocket-reader.ts](file:///home/odmen/DEV/voxlibris/server/websocket-reader.ts))

Специализированный WebSocket для чтения:

- Обработка команд чтения
- Синхронизация позиций между устройствами
- Отправка данных о прогрессе

## Архитектура системы чтения

### Типы сессий

Система поддерживает несколько типов сессий чтения:

1. **Индивидуальные сессии**:
   - Только для одного пользователя
   - Личный прогресс чтения
   - Закладки и заметки

2. **Клубные сессии**:
   - Совместное чтение с другими участниками клуба
   - Синхронизация прогресса
   - Возможность обсуждения во время чтения

3. **Профессиональные сессии**:
   - Чтение профессиональным чтецом
   - Синхронизация с аудио
   - Возможность задавать вопросы

### Компоненты сессии

Каждая сессия чтения включает:

1. **Прогресс чтения**:
   - Текущая позиция (страница, глава, процент)
   - Время начала и продолжительность
   - Скорость чтения

2. **Участники**:
   - Список подключенных пользователей
   - Их текущий прогресс
   - Статус (читает, на паузе, вышел)

3. **Управление**:
   - Возможности воспроизведения/паузы
   - Перемотка вперед/назад
   - Изменение скорости чтения

## Основные функции

### Создание сессии

1. Проверка прав пользователя (владелец книги или участник клуба)
2. Валидация параметров сессии
3. Создание записи сессии в базе данных
4. Назначение начальных параметров
5. Открытие WebSocket-канала для синхронизации

### Присоединение к сессии

1. Проверка прав доступа к сессии
2. Добавление пользователя к списку участников
3. Отправка начального состояния сессии
4. Подключение к WebSocket-каналу сессии

### Синхронизация прогресса

Система обеспечивает синхронизацию в реальном времени:

1. **Отправка обновлений**:
   - Каждые N секунд отправляется текущий прогресс
   - При перемещении по книге
   - При изменении скорости чтения

2. **Получение обновлений**:
   - От других участников сессии
   - От модератора/ведущего
   - От системы (автоматические обновления)

### Управление воспроизведением

Для модераторов/ведущих доступны функции управления:

- Воспроизведение/пауза для всех участников
- Перемотка до определенной позиции
- Управление скоростью чтения
- Остановка сессии

## Интеграция с другими модулями

### Система аутентификации

- Проверка прав доступа к сессиям
- Аутентификация через JWT-токены
- Проверка статуса активности пользователя

### Система клубов

- Ограничение доступа к клубным сессиям
- Синхронизация прогресса среди участников
- Интеграция с расписанием клуба

### Система книг

- Предоставление содержимого книги для чтения
- Синхронизация по главам и страницам
- Обновление статистики чтения книги

### Система коммуникации

- Интеграция с чатом сессии
- Возможность отправки реакций
- Система вопросов и ответов

## Безопасность

- Проверка прав доступа к сессиям
- Защита от несанкционированного управления
- Ограничение частоты отправки обновлений
- Валидация всех данных прогресса

## Возможные проблемы и решения

### Проблемы с синхронизацией

- **Разница во времени**: использование относительных временных меток
- **Потеря соединения**: восстановление состояния при повторном подключении
- **Разное содержание книги**: проверка идентификатора и версии книги

### Проблемы с производительностью

- **Большое количество участников**: эффективная рассылка обновлений
- **Частые обновления прогресса**: буферизация и объединение обновлений
- **Сетевые задержки**: оптимизация протокола общения

### Проблемы с масштабированием

- **Множество одновременных сессий**: распределение по серверам
- **Долгие сессии**: управление памятью и соединениями
- **Отслеживание активности**: эффективные таймеры и проверки

## Тестирование

Модуль чтения и сессий должен быть протестирован на:

- Корректность синхронизации прогресса
- Обработку подключения/отключения участников
- Безопасность управления сессией
- Работу в условиях плохого соединения
- Интеграцию с другими частями приложения
- Обработку граничных случаев (одинаковые позиции, конфликты обновлений и т.д.)
