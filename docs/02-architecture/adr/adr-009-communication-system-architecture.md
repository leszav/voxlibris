# ADR-009: Архитектура системы коммуникации и взаимодействия

## Название

Использование комбинированного подхода с WebSocket для чата и API для реакций и вопросов в системе коммуникации

## Статус

Принято

## Контекст

В платформе VoxLibris необходимо реализовать систему коммуникации и взаимодействия между участниками сессий чтения и клубов. Система должна поддерживать:

- Чат в реальном времени во время сессий
- Систему реакций для быстрой обратной связи
- Систему вопросов для формальных запросов
- Модерацию контента
- Интеграцию с системой аутентификации
- Поддержку различных типов чатов (сессионные, клубные, персональные)

Были рассмотрены следующие альтернативы:
- Только WebSocket для всего функционала
- Только REST API
- Комбинированный подход с WebSocket для чата и API для других функций
- Использование WebRTC для прямой коммуникации

## Решение

Мы выбираем комбинированный подход с использованием WebSocket для чата в реальном времени и REST API для реакций и вопросов по следующим причинам:

1. **Разделение ответственности**: Разные типы коммуникации требуют разных подходов по latency и надежности.

2. **Эффективность**: WebSocket оптимален для чата в реальном времени, а REST API лучше подходит для реакций и вопросов, где важна надежность доставки.

3. **Масштабируемость**: Разные компоненты можно масштабировать независимо.

4. **Надежность**: REST API обеспечивает надежность для важных сообщений (вопросы, реакции), в то время как WebSocket подходит для менее критичного чата.

5. **Интеграция**: Подход позволяет интегрировать с системой уведомлений и другими компонентами.

## Последствия

### Положительные эффекты

- Оптимальная архитектура для разных типов взаимодействия
- Высокая производительность чата в реальном времени
- Надежная доставка важных сообщений (вопросы, реакции)
- Возможность независимого масштабирования компонентов

### Негативные эффекты

- Увеличенная сложность архитектуры
- Необходимость управления двумя различными системами связи
- Потребность в согласовании состояния между компонентами

### Отказ от других альтернатив

- **Только WebSocket**: Отклонен из-за возможных потерь данных при разрыве соединения
- **Только REST API**: Отклонен из-за высокой задержки для чата в реальном времени
- **WebRTC**: Отклонен из-за сложности реализации и несоответствия сценариям использования
