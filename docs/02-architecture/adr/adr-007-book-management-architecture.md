# ADR-007: Архитектура системы управления книгами

## Название

Использование многоуровневой архитектуры с поддержкой различных форматов и безопасной загрузкой для системы управления книгами

## Статус

Принято

## Контекст

В платформе VoxLibris необходимо реализовать систему управления книгами, которая позволяет пользователям загружать, хранить и управлять книгами в различных форматах. Система должна поддерживать:

- Поддержка форматов EPUB и FB2
- Безопасная загрузка файлов с проверкой
- Хранение в S3-совместимом хранилище
- Извлечение метаданных
- Управление доступом к книгам
- Интеграция с системой клубов

Были рассмотрены следующие альтернативы:
- Простое хранение файлов на сервере
- Хранение в базе данных
- Хранение в S3-совместимом хранилище с безопасной проверкой
- Поддержка только одного формата

## Решение

Мы выбираем архитектуру с хранением в S3-совместимом хранилище и многоуровневой проверкой безопасности по следующим причинам:

1. **Масштабируемость**: S3-совместимое хранилище позволяет легко масштабировать систему.

2. **Безопасность**: Многоуровневая проверка (расширение, MIME-тип, magic numbers, размер) защищает от загрузки вредоносных файлов.

3. **Поддержка форматов**: Поддержка EPUB и FB2 охватывает основные форматы электронных книг.

4. **Производительность**: Хранение файлов отдельно от приложения улучшает производительность.

5. **Интеграция**: Архитектура позволяет легко интегрировать с системой клубов и управления доступом.

## Последствия

### Положительные эффекты

- Масштабируемое решение для хранения книг
- Высокий уровень безопасности при загрузке
- Поддержка наиболее популярных форматов
- Легкая интеграция с другими системами
- Возможность использования CDN для ускорения доступа

### Негативные эффекты

- Усложнение архитектуры из-за необходимости интеграции с внешним хранилищем
- Зависимость от внешней инфраструктуры
- Необходимость управления URL-адресами доступа к файлам

### Отказ от других альтернатив

- **Хранение файлов на сервере**: Отклонено из-за проблем с масштабируемостью
- **Хранение в базе данных**: Отклонено из-за снижения производительности
- **Поддержка только одного формата**: Отклонено из-за ограничения возможностей пользователей
