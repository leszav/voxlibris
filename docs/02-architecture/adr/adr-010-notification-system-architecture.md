# ADR-010: Архитектура системы уведомлений и почты

## Название

Использование многоуровневой архитектуры с поддержкой различных каналов доставки для системы уведомлений

## Статус

Принято

## Контекст

В платформе VoxLibris необходимо реализовать систему уведомлений и почты, обеспечивающую доставку информации пользователям через различные каналы. Система должна поддерживать:

- Email-уведомления для важных событий
- Push-уведомления для интерактивных событий
- Внутренние уведомления в приложении
- Настройку предпочтений пользователя
- Интеграцию с другими системами (аутентификация, клубы, чтение)
- Защиту от спама и перегрузки пользователя уведомлениями

Были рассмотрены следующие альтернативы:
- Только email-уведомления
- Только push-уведомления
- Только внутренние уведомления
- Комбинированная система с настройками пользователя

## Решение

Мы выбираем комбинированную систему уведомлений с поддержкой нескольких каналов и настройками пользователя по следующим причинам:

1. **Гибкость**: Разные пользователи предпочитают разные каналы получения уведомлений.

2. **Надежность**: Многоуровневая система обеспечивает доставку даже при недоступности одного из каналов.

3. **UX**: Пользователи могут настроить получение только тех уведомлений, которые им нужны.

4. **Интеграция**: Архитектура позволяет легко интегрировать с различными системами платформы.

5. **Аналитика**: Возможность отслеживания эффективности разных каналов.

## Последствия

### Положительные эффекты

- Персонализированный опыт для пользователей
- Высокая вероятность доставки важной информации
- Гибкость в настройке уведомлений
- Возможность анализа эффективности разных каналов

### Негативные эффекты

- Усложнение архитектуры системы
- Необходимость управления несколькими каналами доставки
- Потребность в системе управления настройками пользователей
- Возможность перегрузки пользователя уведомлениями при неправильной настройке

### Отказ от других альтернатив

- **Только email**: Отклонен из-за медленной доставки и зависимости от почтовых клиентов
- **Только push**: Отклонен из-за ограничений браузеров и мобильных платформ
- **Только внутренние**: Отклонен из-за возможной пропуска важной информации
