# ADR-001: Выбор Drizzle ORM для взаимодействия с базой данных

## Название

Выбор Drizzle ORM для взаимодействия с PostgreSQL базой данных

## Статус

Принято

## Контекст

Нам нужно выбрать ORM (объектно-реляционное отображение) или библиотеку для взаимодействия с PostgreSQL базой данных. В проекте используется TypeScript, и нам нужен инструмент, который обеспечит:

- Строгую типизацию при работе с базой данных
- Хорошую производительность
- Совместимость с PostgreSQL
- Поддержку миграций
- Легкость тестирования
- Совместимость с архитектурным паттерном Repository

Были рассмотрены такие варианты как Prisma, TypeORM, Sequelize, Knex.js и Drizzle ORM.

## Решение

Мы выбираем Drizzle ORM для взаимодействия с базой данных по следующим причинам:

1. **Строгая типизация**: Drizzle ORM предлагает отличную поддержку TypeScript, включая вывод типов на основе схемы БД.

2. **Производительность**: Drizzle ORM - это легковесное решение без runtime overhead, которое генерирует эффективные SQL-запросы.

3. **Прозрачность**: В отличие от Prisma с его автогенерацией, Drizzle позволяет разработчику лучше понимать и контролировать SQL-запросы.

4. **Поддержка миграций**: Drizzle имеет встроенный инструментарий для миграций, хорошо интегрированный с workflow проекта.

5. **Активное развитие**: Проект активно развивается и имеет хорошую поддержку сообщества.

6. **Совместимость с Repository паттерном**: Drizzle легко интегрируется с паттерном Repository, который мы используем в архитектуре.

## Последствия

### Положительные эффекты

- Улучшенная типобезопасность при работе с базой данных
- Более предсказуемая производительность SQL-запросов
- Упрощенная отладка запросов
- Лучшая совместимость с архитектурным паттерном Repository
- Упрощенная поддержка миграций

### Негативные эффекты

- Необходимость писать больше boilerplate кода по сравнению с Prisma
- Необходимость более глубокого знания SQL
- Новый инструмент для команды, требующий обучения

### Отказ от других альтернатив

- **Prisma**: Отклонен из-за runtime зависимости и меньшего контроля над запросами
- **TypeORM**: Отклонен из-за менее стабильного опыта работы с типами в TypeScript
- **Sequelize**: Отклонен из-за отсутствия нативной поддержки TypeScript
- **Knex.js**: Отклонен из-за отсутствия встроенной типизации
