# ADR-008: Архитектура системы чтения и сессий

## Название

Использование WebSocket для синхронизации сессий чтения в реальном времени с поддержкой различных типов сессий

## Статус

Принято

## Контекст

В платформе VoxLibris необходимо реализовать систему чтения и сессий, обеспечивающую синхронизацию прогресса чтения, управление сессиями и взаимодействие между участниками. Система должна поддерживать:

- Индивидуальные сессии чтения
- Клубные сессии с синхронизацией
- Профессиональные сессии с аудио
- Синхронизацию прогресса в реальном времени
- Управление воспроизведением
- Интеграцию с системой аутентификации и клубов

Были рассмотрены следующие альтернативы:
- Опрос сервера (polling)
- Server-Sent Events
- WebSocket с использованием Socket.IO
- WebRTC для peer-to-peer синхронизации

## Решение

Мы выбираем WebSocket с использованием Socket.IO для синхронизации сессий чтения по следующим причинам:

1. **Низкая задержка**: WebSocket обеспечивает минимальную задержку для синхронизации, что критично для совместного чтения.

2. **Двунаправленная связь**: В отличие от SSE, WebSocket позволяет передавать данные в обе стороны, что необходимо для управления воспроизведением и чатом.

3. **Поддержка комнат**: Socket.IO предоставляет встроенную поддержку комнат, что идеально подходит для сессий с ограниченным числом участников.

4. **Надежность**: Socket.IO включает механизмы восстановления соединения и fallback-механизмы.

5. **Интеграция**: Хорошо интегрируется с существующим стеком (Express.js, React).

## Последствия

### Положительные эффекты

- Реальная синхронизация прогресса чтения между участниками
- Поддержка интерактивного чата во время сессии
- Низкая задержка для управления воспроизведением
- Поддержка различных типов сессий

### Негативные эффекты

- Необходимость управления долгосрочными соединениями на сервере
- Увеличение сложности при масштабировании
- Потребность в обработке отключения клиентов и восстановления соединений
- Увеличение потребления ресурсов по сравнению с HTTP

### Отказ от других альтернатив

- **Polling**: Отклонен из-за высокой нагрузки на сервер и задержек
- **Server-Sent Events**: Отклонен из-за односторонней связи (только сервер → клиент)
- **WebRTC**: Отклонен из-за сложности реализации и несоответствия сценариям использования
