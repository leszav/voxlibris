# ADR-004: Выбор React в качестве фронтенд-фреймворка

## Название

Выбор React в качестве основного фронтенд-фреймворка для пользовательского интерфейса

## Статус

Принято

## Контекст

Для реализации пользовательского интерфейса VoxLibris нужно выбрать фронтенд-фреймворк. Требования к фреймворку:

- Поддержка компонентного подхода
- Хорошая производительность (виртуальный DOM)
- Большое сообщество и экосистема
- Поддержка TypeScript
- Совместимость с существующими инструментами (Vite, Tailwind CSS)
- Возможность реализации сложных интерфейсов (аудиоплеер, чат, синхронизация прогресса)
- Возможность тестирования компонентов

Были рассмотрены следующие альтернативы:
- React
- Vue.js
- Angular
- Svelte/SvelteKit
- Vanilla JavaScript с Web Components

## Решение

Мы выбираем React в качестве основного фронтенд-фреймворка по следующим причинам:

1. **Большое сообщество**: React имеет самое большое сообщество среди фронтенд-фреймворков, что обеспечивает хорошую поддержку и множество готовых решений.

2. **Экосистема**: Большое количество готовых библиотек, компонентов и инструментов, совместимых с React.

3. **Поддержка TypeScript**: Отличная поддержка TypeScript, что соответствует нашему требованию к строгой типизации.

4. **Производительность**: Виртуальный DOM и оптимизации React обеспечивают хорошую производительность даже при сложных интерфейсах.

5. **Опыт команды**: Ожидается, что у большинства разработчиков уже есть опыт работы с React.

6. **Поддержка компонентного подхода**: Отличная поддержка компонентного подхода, что соответствует архитектурным принципам проекта.

7. **Инструменты разработки**: Отличные инструменты разработки (React DevTools) для отладки и профилирования.

## Последствия

### Положительные эффекты

- Быстрая разработка благодаря готовым компонентам и библиотекам
- Хорошая поддержка и документация
- Легкость найма разработчиков с опытом React
- Совместимость с современными инструментами сборки (Vite)
- Возможность повторного использования компонентов

### Негативные эффекты

- Потенциально более высокая сложность для новичков
- Необходимость управления состоянием (хотя с появлением React 19 это упрощается)
- Размер финального бандла может быть больше, чем у некоторых альтернатив

### Отказ от других альтернатив

- **Vue.js**: Отклонен из-за меньшего сообщества и экосистемы по сравнению с React
- **Angular**: Отклонен из-за более жесткой архитектуры и более крутой кривой обучения
- **Svelte/SvelteKit**: Отклонен из-за меньшей зрелости экосистемы и меньшего количества готовых решений
- **Vanilla JavaScript**: Отклонен из-за отсутствия абстракций для сложных интерфейсов и необходимости изобретать велосипеды
