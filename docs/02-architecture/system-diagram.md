## Подробное описание компонентов

### Frontend (Client)

**Presentation Layer**:
- **Components**: React компоненты для пользовательского интерфейса
- **Pages**: Страницы приложения и роутинг
- **State Management**: React Query для управления состоянием и кэширования

### Backend (Server)

**API Layer**:
- **Authentication Endpoints**: Регистрация, вход, восстановление пароля
- **Club Endpoints**: Управление клубами и участниками
- **Book Endpoints**: Загрузка и управление книгами
- **Reading Session Endpoints**: Управление сессиями чтения
- **Communication Endpoints**: Чат, реакции, вопросы

**Business Logic Layer**:
- **Club Service**: Логика управления клубами
- **Book Service**: Обработка книг и метаданных
- **Reading Service**: Управление сессиями чтения
- **Notification Service**: Управление уведомлениями
- **Email Service**: Отправка email-уведомлений
- **Analytics Service**: Сбор и обработка аналитики

**Infrastructure Layer**:
- **Authentication Middleware**: JWT проверка и авторизация
- **Rate Limiting Middleware**: Защита от перегрузки
- **Logging Middleware**: Логирование запросов
- **Security Middleware**: Защита от атак

**Real-time Interface**:
- **WebSocket Handlers**: Обработка синхронизации чтения
- **Chat System**: Реальное время общения
- **Progress Sync**: Синхронизация прогресса чтения

### Database & Storage Layer

**Relational Database**:
- **PostgreSQL**: Хранение структурированных данных
- **Drizzle ORM**: Объектно-реляционное отображение

**File Storage**:
- **S3 Compatible Storage**: Хранение книг, обложек и аудиозаписей
- **MinIO**: Локальное S3-совместимое хранилище

**Caching & Session**:
- **Redis**: Хранение сессий и ограничений по частоте запросов

**External Services**:
- **SMTP Server**: Отправка email-уведомлений

## Взаимодействие компонентов

1. **Клиент ↔ Сервер**: HTTP/HTTPS и WebSocket соединения
2. **Сервер ↔ База данных**: Прямое подключение через Drizzle
3. **Сервер ↔ Файловое хранилище**: S3 протокол
4. **Сервер ↔ Redis**: Для сессий и ограничений
5. **Сервер ↔ SMTP**: Для отправки уведомлений

## Безопасность в архитектуре

- **JWT токены**: Stateless аутентификация
- **Rate limiting**: Защита от DDoS через Redis
- **Input validation**: Проверка через Zod
- **File security**: Проверка типов и размеров файлов
- **CORS**: Ограничение доступа с разных доменов
- **Helmet**: Защита HTTP заголовков
