# Система клубов и сообществ

## Обзор

Модуль системы клубов и сообществ - это ключевая часть приложения VoxLibris, позволяющая пользователям создавать, управлять и участвовать в книжных клубах. Система поддерживает различные типы клубов, механизмы приглашений, роли участников и функции обсуждения книг.

## Компоненты системы

### 1. Club Routes ([server/club-routes.ts](file:///home/odmen/DEV/voxlibris/server/club-routes.ts))

Основной файл, содержащий маршруты для управления клубами:

- `/api/clubs` - создание и получение списка клубов
- `/api/clubs/:id` - получение, обновление и удаление конкретного клуба
- `/api/clubs/:id/members` - управление участниками клуба
- `/api/clubs/:id/invite` - создание приглашений в клуб
- `/api/clubs/:id/roles` - управление ролями участников
- `/api/user/clubs` - получение клубов, в которых состоит пользователь

### 2. Club Reader Routes ([server/club-reader-routes.ts](file:///home/odmen/DEV/voxlibris/server/club-reader-routes.ts))

Маршруты, связанные с чтением в клубах:

- `/api/clubs/:id/current-book` - установка и получение текущей книги клуба
- `/api/clubs/:id/reading-progress` - отслеживание прогресса чтения
- `/api/clubs/:id/bookmarks` - управление закладками в клубе
- `/api/clubs/:id/notes` - заметки участников

### 3. Club Discussions Routes ([server/club-discussions-routes.ts](file:///home/odmen/DEV/voxlibris/server/club-discussions-routes.ts))

Маршруты для обсуждений в клубах:

- `/api/clubs/:id/discussions` - создание и получение обсуждений
- `/api/clubs/:id/discussions/:discussionId` - получение и обновление конкретного обсуждения
- `/api/clubs/:id/discussions/:discussionId/comments` - комментарии к обсуждениям

## Архитектура системы клубов

### Типы клубов

Система поддерживает различные типы клубов:

1. **Открытые клубы**:
   - Любой пользователь может присоединиться без приглашения
   - Публичный список участников
   - Доступ к материалам клуба ограничен

2. **Закрытые клубы**:
   - Требуется приглашение для присоединения
   - Приватный список участников
   - Полный доступ к материалам клуба

3. **Клубы с ограниченным доступом**:
   - Ограниченное количество участников
   - Приватный режим с приглашениями
   - Дополнительные права модерации

### Роли участников

Каждый участник клуба может иметь одну из следующих ролей:

1. **Владелец (Owner)**:
   - Полный контроль над клубом
   - Может удалять клуб
   - Может изменять любые настройки
   - Автоматически становится модератором

2. **Модератор (Moderator)**:
   - Управление участниками
   - Модерация контента
   - Контроль за обсуждениями
   - Управление расписанием

3. **Участник (Member)**:
   - Участие в чтении
   - Создание заметок и закладок
   - Участие в обсуждениях
   - Присоединение к сессиям чтения

4. **Гость (Guest)**:
   - Ограниченный доступ к контенту
   - Просмотр расписания
   - Только чтение обсуждений

## Основные функции

### Создание клуба

1. Проверка прав пользователя (должен быть аутентифицирован)
2. Валидация данных с использованием Zod
3. Создание записи клуба в базе данных
4. Назначение пользователя владельцем клуба
5. Установка начальных настроек клуба

### Присоединение к клубу

Существует несколько способов присоединения:

1. **По приглашению**:
   - Получение токена приглашения
   - Проверка срока действия приглашения
   - Проверка соответствия email
   - Добавление пользователя в клуб

2. **Через поиск**:
   - Проверка типа клуба (открытый/закрытый)
   - Проверка вместимости
   - Добавление пользователя в клуб

3. **По запросу**:
   - Отправка запроса владельцу
   - Рассмотрение запроса модератором
   - Принятие решения о принятии

### Управление участниками

- Добавление и удаление участников
- Назначение и изменение ролей
- Блокировка и разблокировка участников
- Отображение статистики участия

### Приглашения

Система приглашений включает:

1. **Создание приглашений**:
   - Генерация уникального токена
   - Установка срока действия
   - Отправка email с приглашением

2. **Обработка приглашений**:
   - Проверка токена
   - Проверка срока действия
   - Присоединение к клубу
   - Обновление статуса приглашения

## Интеграция с другими модулями

### Система аутентификации

- Проверка аутентификации пользователей
- Проверка прав доступа к клубам
- Управление ролями участников

### Система книг

- Управление текущей книгой клуба
- Доступ к книгам только для участников
- Совместное чтение книг

### Система чтения

- Сессии чтения в клубе
- Синхронизация прогресса участников
- Совместные закладки и заметки

### Система уведомлений

- Уведомления о приглашениях
- Уведомления о новых обсуждениях
- Уведомления о сессиях чтения

## Безопасность

- Проверка прав доступа ко всем операциям
- Защита от несанкционированного доступа к данным клубов
- Ограничение количества участников
- Защита от спама в обсуждениях

## Возможные проблемы и решения

### Проблемы с приглашениями

- **Истечение срока действия**: автоматическая проверка и обновление статуса
- **Несоответствие email**: проверка соответствия при принятии приглашения
- **Множественное использование**: одноразовое использование токенов

### Проблемы с масштабированием

- **Большое количество участников**: эффективные запросы к базе данных
- **Множество обсуждений**: пагинация и фильтрация
- **Синхронизация прогресса**: оптимизация WebSocket-соединений

## Тестирование

Модуль клубов должен быть протестирован на:

- Корректность создания и управления клубами
- Безопасность доступа к закрытым клубам
- Работу системы приглашений
- Управление ролями участников
- Интеграцию с другими частями приложения
- Обработку граничных случаев (переполнение, удаление владельца и т.д.)
